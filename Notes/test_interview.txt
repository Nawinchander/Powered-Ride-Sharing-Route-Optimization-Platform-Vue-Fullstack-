Interview Questions
Unit Test

Q: What is a Unit Test?

A: Tests a single function or method in isolation without external dependencies.

Integration Test

Q: What is an Integration Test?

A: Verifies that multiple modules work together correctly, such as an API endpoint interacting with the database.

End-to-End (E2E) Test

Q: What is an E2E Test?

A: Simulates real user behavior by testing the entire application flow from the UI to the backend.

Jest vs Vitest
Jest	Vitest
Common for Node.js backend	Optimized for Vite and Vue frontend
Great for Express APIs	Fast component testing
Supertest

Q: Why use Supertest?

A: It allows you to test Express API endpoints without manually running HTTP clients.


Cypress

Q: Why use Cypress?

A: It automates browser interactions, making it ideal for testing complete user workflows like logging in and booking a ride.

These tests are intentionally simple but demonstrate the testing structure and tools that interviewers expect to see in a portfolio project. As your project grows, you can replace these starter tests with more comprehensive assertions and mocks.
