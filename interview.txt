Interview Questions
Why hash passwords?

Passwords should never be stored in plaintext because database breaches would expose user credentials.

Why JWT instead of sessions?

JWT scales better in distributed systems because the server does not need to maintain session state.

Why Repository Pattern?

Repositories separate business logic from database logic which improves testability and maintainability.

Interview Questions
Authentication
Why use Access Token and Refresh Token together?

Access tokens should be short-lived for security, while refresh tokens allow seamless re-authentication without forcing users to log in repeatedly.

Why not store JWT in Local Storage?

Local storage is vulnerable to XSS attacks. Secure HTTP-only cookies are generally safer for production applications.

Why RBAC instead of hardcoded permissions?

Role-based access control scales much better as applications grow and more user roles are introduced.

Why use WebSockets instead of polling?

Polling sends repeated requests regardless of whether data changes, while WebSockets provide real-time bidirectional communication with lower latency and lower server load.

Why store driver locations in Redis instead of MongoDB?

Driver locations update every few seconds. Redis handles high-frequency writes and reads significantly better than disk-based databases.

How would you scale Socket servers horizontally?

Using the Redis Pub/Sub adapter with multiple Socket.IO instances behind a load balancer.

Composables allow reusable business logic across multiple components without inheritance or mixins.

Why use a store instead of component state?

Global application state such as authentication and tracking data should remain centralized to avoid prop drilling and inconsistent state.

Interview Questions
Why use a Service Layer?

Business logic should not live in controllers because it makes code harder to maintain and test.

Why use a Repository Layer?

Repositories isolate database operations and allow easy migration from MongoDB to another database later.

Why use WebSockets for booking updates?

Ride status changes happen in real time. Polling every few seconds creates unnecessary requests and increases infrastructure costs.

Why separate API, Store and Components in the frontend?
API layer → network communication
Store layer → application state
Components → presentation logic

This separation improves scalability and testability.
